About

A research-grade platform built for transparency.

MediAI is an academic capstone exploring what responsible, integrated artificial intelligence looks like when applied to the most common healthcare questions a curious user can ask.

Our Mission

Closing the gap between curiosity and credible information.

Healthcare is the most consequential domain in which artificial intelligence is deployed today, and also the most exacting. The cost of a confidently wrong answer can be measured in lives. We believe the appropriate response to this reality is not to avoid the domain but to engineer for it — with transparency, conservative scoping, and explicit safety mechanisms layered above every probabilistic component.

MediAI is a deliberate exercise in restraint. Each of its five services addresses a single well-defined question. Each model is selected only after rigorous benchmarking. Outputs are kept clear, calibrated where applicable, and paired with an unambiguous disclaimer reminding the user that the platform is a starting point for informed conversation with a clinician — not a substitute for one.

The Argument

Why an integrated platform.

Most existing healthcare AI tools are single-purpose: a symptom checker over here, a calorie counter over there, an image classifier somewhere else. A user with overlapping concerns must navigate five interfaces, re-enter five sets of context, and reconcile the outputs unaided. MediAI consolidates these concerns under a unified contract — the same look, the same API style, the same disclaimers, the same privacy posture.

The argument is one of friction, not of capability. Each of our individual services could be improved by a dedicated team; together, they offer something none of them does alone — a coherent first hour for the curious patient.

Technology

Built on open, modern infrastructure.

Every component is open-source, freely available, and chosen with a preference for explainability over novelty.

Backend

Python 3 · Flask · scikit-learn · TensorFlow · Keras · joblib

ML Stack

Random Forest · Gradient Boosting · CNN with MobileNetV2 backbone · Sentence-Transformers · FAISS

Frontend

HTML5 · Bootstrap 5 · Bootstrap Icons · Vanilla JavaScript · Inter typeface

Datasets

Kaggle public datasets: CBC, Symptom-Disease, Chest X-Ray Pneumonia, HAM10000 · NIMH and WHO reference documents

Process

From idea to deployed platform.

A four-stage development process, each stage building on the artefacts of the last.

PHASE 01
Ideation

Identify the gap in current AI healthcare tooling; define a focused, defensible scope.

PHASE 02
Data & EDA

Source public datasets, clean, visualise distributions, document patterns and biases.

PHASE 03
Modelling

Benchmark candidates, tune hyperparameters, serialise the chosen model for deployment.

PHASE 04
Integration

Expose each model through a uniform REST API behind a single coherent frontend.

Vision

What we hope this becomes.

A reference implementation. A teaching artefact for the next cohort of students studying applied machine learning. A starting point for adaptations into adjacent domains — veterinary medicine, agricultural diagnostics, public-health epidemiology. The codebase is permissively licensed precisely so that it can be extended without negotiation.

Limitations

What it is not.

Not a medical device. Not a substitute for a clinician. Not certified by any regulatory authority. Not validated on demographically representative cohorts. Not designed to handle protected health information. The platform's value is informational and pedagogical; any claim beyond that would be premature.

Curious about the source code?

Every line of this platform — backend, frontend, model training scripts, even the documentation — is open for inspection.
